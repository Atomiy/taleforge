## 1. 环境准备与启动

### 1.1 前置条件

| 项目 | 要求 | 检查结果（用户填写） |
|------|------|:-------------------:|
| Python 版本 | >= 3.10 | |
| 操作系统 | Windows / Linux / macOS | |
| 网络连接 | 可访问 `api.deepseek.com` | |
| DeepSeek API Key | 有效的 API Key | |

### 1.2 启动步骤

#### 步骤 1：激活 Python 虚拟环境

打开终端（命令提示符或 PowerShell），进入项目根目录，执行：

```bash
# Windows PowerShell
.\venv\Scripts\Activate.ps1

# Windows CMD
.\venv\Scripts\activate.bat

# Linux / macOS
source venv/bin/activate
```

激活成功后，终端提示符前会出现 `(venv)` 标识。

#### 步骤 2：安装依赖包

```bash
pip install -r requirements.txt
```

> 依赖清单：fastapi、uvicorn、pydantic、requests、python-dotenv、sseclient-py

#### 步骤 3：启动服务器

```bash
python start.py
```

启动成功后，控制台应显示类似以下信息：

```
╔═══════════════════════════════════════════╗
║         TaleForge - AI Story Generator    ║
╚═══════════════════════════════════════════╝

  ║  Configuration:
  ║    HOST  = 127.0.0.1
  ║    PORT  = 8080
  ║  Port 8080 is available
  ║  Starting TaleForge server...
  ║  URL: http://127.0.0.1:8080
```

> **注意**：若端口 8080 被占用，启动脚本会自动尝试释放端口。

#### 步骤 4：访问系统

打开浏览器，访问 **http://127.0.0.1:8080**。

页面应正常渲染，标题显示为 **TaleForge - 智能故事生成器**，导航栏包含「创作」「作品」「历史」「设置」四个按钮。

#### 步骤 5：配置 API Key

1. 点击导航栏右侧的「设置」按钮。
2. 在弹出的对话框中输入您的 DeepSeek API Key。
3. 点击「保存设置」，页面右上角应出现绿色提示「API Key 保存成功」。
4. 关闭设置对话框。

> **获取 API Key**：访问 [https://platform.deepseek.com/](https://platform.deepseek.com/) 注册并创建 API Key。

## 3. 用户操作流程指引

### 3.1 完整工作流：创建作品 → 规划卷 → 批量生成 → 查看结果

此流程覆盖 TaleForge 的核心创作链路，建议用户按以下顺序完整走一遍。

```
创建作品 → 添加卷 → 规划卷（大纲/角色/伏笔） → 跨章角色出场表 → 一键生成全卷 → 查看章节内容 → 生成作品概要
```

#### Step 1：创建作品

1. 进入「作品」标签页。
2. 填写「作品名称」、选择「体裁」和「风格」。
3. 填写「世界观设定」和「故事大纲」。
4. 点击「创建作品」。
5. 系统自动打开新创建的作品详情弹窗。

#### Step 2：进入卷规划

1. 默认第一卷已自动创建，点击卷标题展开。
2. 点击「规划」标签切换到规划视图。

#### Step 3：填写卷规划信息

填写以下内容（越详细越好，这直接影响生成质量）：

| 字段 | 说明 | 示例 |
|------|------|------|
| 卷职责 | 本卷在整部作品中的功能定位 | 引入核心冲突，建立世界观，主角初次冒险 |
| 卷大纲 | 本卷的故事脉络 | 主角在废弃矿坑中发现古老遗迹，意外激活封印...... |
| 与上卷衔接 | 上一卷结束时的情况 | 这是第一卷，无需填写 |
| 对下卷铺垫 | 为下一卷埋下线索 | 遗迹深处传来低语声，暗示更大危机即将到来 |
| 出场人物 | 本卷涉及的所有角色 | 林深（考古系学生）、陈霜（神秘学专家） |
| 伏笔清单 | 本卷需要埋下的伏笔 | 矿坑墙壁上刻满同样的符号 |

#### Step 4：配置跨章角色出场表

1. 在规划页面向下滑动到「跨章人物出场表」区域。
2. 为每一章勾选出场的角色。
3. 使用「一键填充」快速将所有角色分配到所有章节。
4. 观察校验警告——确保没有角色被遗漏。

> **三种角色出场类型说明**：
> - **主动**（默认）：角色在该章主动行动、推动剧情。
> - **被动**：角色在该章被提及、受影响，但不主动行动。
> - **缺席**：角色在该章不出场（不勾选即缺席）。

在规划面板中，系统会自动根据勾选情况推断角色出场目的：
- 主动角色 → 「推进剧情」
- 被动角色 → 「塑造人物」「表达主题」「营造氛围」

#### Step 5：保存规划

点击「保存规划」按钮，确保所有规划数据已持久化。

#### Step 6：一键生成全卷

1. 点击卷头部的**魔棒图标**（一键生成全卷按钮）。
2. 如果该卷已有章节，系统会弹出确认覆盖提示。
3. 确认后，系统自动切换到创作页面，开始逐章生成。
4. 生成过程中，屏幕中央显示进度覆盖层，包含当前章节序号和进度条。
5. 系统自动为每章填充对应的角色（仅勾选的角色）、伏笔，并保持章节间续写连贯。
6. 全部生成完毕后，自动跳回作品详情页面。

#### Step 7：查看章节内容

1. 在作品详情中展开卷。
2. 点击任意章节的「查看」按钮，在预览区域查看故事内容。
3. 点击「编辑」可打开编辑弹窗修改内容。
4. 确认各章内容连贯，角色出场符合出场表配置。

#### Step 8：生成作品概要

1. 在作品详情弹窗底部，点击「生成概要」按钮。
2. 系统 AI 会根据各章节内容自动生成一段作品概要。
3. 概要将显示在作品详情页的「内容概要」区域。

### 3.2 约束系统使用指南

TaleForge 的约束系统通过「卷规划」面板中的各类字段实现，用于引导 AI 生成符合预期的故事内容。

#### 3.2.1 各类约束的作用方式

| 约束类型 | 配置位置 | 作用说明 |
|---------|---------|---------|
| **角色约束** | 规划 → 本卷出场人物 | 限定生成的角色池，AI 只能使用这些角色 |
| **出场约束** | 规划 → 跨章人物出场表 | 限定每章出场的角色，AI 不会让不该出现的角色出场 |
| **伏笔约束** | 规划 → 本卷伏笔清单 | 要求故事中包含这些伏笔元素 |
| **大纲约束** | 规划 → 卷故事大纲 | 限定故事的发展方向和关键事件 |
| **衔接约束** | 规划 → 与上卷/下卷衔接 | 保持跨卷情节的连贯性 |

#### 3.2.2 角色出场约束详述

在跨章角色出场表中，每个角色在每章有三种参与状态：

| 状态 | 界面表现 | 生成效果 |
|:----:|---------|---------|
| **主动** | 紫色高亮勾选 | AI 会让该角色在本章有行动、对话、推动情节 |
| **被动** | 紫色高亮（可通过 role 字段区分） | 角色在本章被提及或受事件影响，但无主动行动 |
| **缺席** | 灰色未勾选 | 角色不出现在本章内容中 |

> 当前版本中，出场表默认所有勾选角色视为「主动」角色。系统会自动为主动角色赋予「推进剧情」的出场目的，为被动角色赋予「塑造人物/表达主题/营造氛围」的出场目的。

#### 3.2.3 如何验证角色合规性

一键生成全卷后，可以通过以下方式验证角色出场是否符合约束：

1. **查看出场表校验警告**：在规划面板中，系统会实时显示：
   - 「角色 X 在出场人物库中但未在任何章节出场」
   - 「角色 Y 出现在出场表中但不在本卷出场人物库中」
2. **阅读章节内容**：手动抽查已生成章节，确认：
   - 未勾选的角色是否确实没有出现
   - 勾选的角色是否按预期行事

### 3.3 常见问题与排查

| 问题现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 生成内容为空或报错 | API Key 未配置或无效 | 检查设置中的 API Key，重新保存 |
| 批量生成中断 | 网络波动或 API 超时 | 重新执行一键生成，已生成的章节不会丢失 |
| 角色未按出场表出现 | 约束信息未正确保存 | 回到规划面板，重新保存后重试 |
| 续写章节内容不连贯 | 前一章信息未正确传递 | 从作品详情点击「续写」按钮进入，确保 previous_story_id 正确 |
| 生成速度慢 | 字数较多或网络延迟 | 适当降低目标字数（2000-3000 字为宜） |
| 端口被占用 | 上次进程未退出 | `python start.py` 会自动清理旧进程 |
