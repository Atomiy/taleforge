# v0.4.0 进度锚点约束

## 发现的问题

对比《苍狼传说》（因果驱动）和《星际余晖》（时间驱动）的章节连贯性后，发现一个比"章节开头跳跃"更严重的问题：**情节倒退（时间回溯/场景重置）**。

具体表现：第3章标题与第2章重复（都是"暗流涌动"），且第3章开篇将剧情"倒带"到了第2章中段的状态，否定了第2章结尾已经发生的剧情推进。

## 根因分析

1. **planner 层**：续写提示只传了上章最后500字作为"内容摘要"，没有明确标注"这是上章结尾，新章必须从这之后开始"
2. **writer 层**：三层控制中的过渡规则只约束了"首句衔接"，没有约束"整章定位"

## 修复方案

### 1. planner.py 改动

续写提示的 `continuation_info` 中：
- 将"内容摘要"拆分为明确的"上章开头（300字）"和"**上章结尾（400字）——新章节必须从这点之后开始**"
- 新增"硬约束：进度锚点"小节，包含4条规则：
  1. 新章起始状态必须 ≥ 上章结尾最后一句的状态
  2. 严禁时间回溯/场景重置（含明确示例：上章结尾坐进逃生舱→本章不能还在指挥室）
  3. 章节结尾必须比开头有明显推进
  4. 新章标题不能与上一章重复

### 2. writer.py 改动

在 `_build_system_context` 和 `_build_prompt` 中新增"规则7 — 进度锚点"：
1. 起始状态必须等于或晚于上章末尾最后一句所述状态
2. 禁止时间回溯/场景重置
3. 结尾必须比开头有明显推进
4. 标题不能重复

## 验证结果

| 指标 | 修复前（旧第3章） | 修复后（新第3章） |
|------|:---:|:---:|
| 标题 | 暗流涌动（重复） | 裂口之秘（不重复） |
| 起始状态 | 场景重置到第2章中段 | 代码拆除87%（接续第2章结尾） |
| 剧情节拍 | 2个（基本原地踏步） | 7个（持续向前推进） |
| 结尾锚点 | 仍在侦察船 | 获得第二信标坐标 |
