#### v0.3.2 分层控制方案

采用"战术层 + 战略层 + 自由层"三层架构：

| 层级 | 内容 | 作用范围 | 约束力度 |
|:-----|:-----|:---------|:---------|
| **战术层（过渡）** | 上一章结尾 **400字** + 4条过渡规则 | 仅控制新章**前200字（第一段）** | **超强约束** |
| **战略层（大纲）** | 完整大纲 + 3条执行规则 | 从新章**第二段开始**至全章 | **最高优先级** |
| **自由层（风格）** | 上一章结尾 **1500字** 完整参考 | 贯穿全章 | **弱参考**，不强制复现 |

过渡规则修正（对比 v0.3.1）：

| v0.3.1（已废弃） | v0.3.2（生效） |
|:-----------------|:---------------|
| 禁止"第二天清晨" | 允许——若上章有入睡/昏迷/约定铺垫 |
| 冲突必须继续吵 | 允许余波（冷暴力/转移话题），不要求继续争吵 |
| 首句必须时空直接衔接 | 第一段末尾通过环境暗示过渡即可 |
| 全章约束 | 仅限第一段（约200字） |

## 技术要点

1. **确认弹窗**：Vue 3 Teleport + Transition，Promise 封装，统一模态层
2. **内联预览**：ref 控制，在作品详情弹窗内插入预览区域，避免弹窗嵌套
3. **状态持久化**：sessionStorage（表单临时状态）+ localStorage（卷折叠偏好）
4. **快捷键**：全局 keydown 事件监听，支持多模态层 ESC 逐层关闭
5. **进度追踪**：基于 SSE agent 事件驱动，4 阶段映射进度百分比
