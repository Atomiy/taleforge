# Phase 3: 交互逻辑优化方案

## 1. 概述

基于对现有前端代码的全面审查和行业设计模式研究（包括 Vue 模态交互规范、创作工具 UX 设计、Toast/Confirm 交互模式），针对 TaleForge 当前的交互问题进行系统优化。

## 3. 优化方案（按迭代优先级）

### 第一优先级：P0（基础体验修复）

#### P3-01: 自定义确认弹窗组件

**目标**：替代所有原生 `confirm()`，提供统一、可定制的确认对话框。

**方案**：
- 使用 Vue 3 Teleport + Transition 构建通用 Modal 组件
- 支持三种类型：`confirm`（确认/取消）、`alert`（仅确认）、`danger`（红色危险操作）
- Promise 化 API：`await Modal.confirm({ title, message, confirmText, cancelText })`

**涉及调用点**：
- `deleteWork()` — 删除作品
- `deleteVolume()` — 删除卷
- `deleteStory()` — 删除故事

**参考设计**：
```javascript
// 使用方式
async function deleteWork(id) {
    const ok = await Modal.confirm({
        title: '删除作品',
        message: '确定删除该作品及其所有章节关联？此操作不可撤销。',
        type: 'danger',
        confirmText: '确认删除'
    })
    if (!ok) return
    // 执行删除...
}
```

#### P3-06: 弹窗层级优化

**问题**：`showStoryDetail` 覆盖 `showWorkDetail`，查看章节时先关闭作品弹窗，关闭故事弹窗后不回退到作品弹窗。

**方案**：
- 章节查看改为**内联展开模式**：在卷树中点击"查看"时，在当前弹窗内展开内容预览区域（slide-down 展开），而非另开新弹窗
- 如需全屏查看，改为在当前弹窗内切换到"阅读模式"（切换内容视图，不关闭原弹窗）

**交互流程**：
```
优化前：
  作品弹窗 → 点击查看章节 → 关闭作品弹窗 → 打开故事弹窗 → 关闭故事弹窗 → 回到列表
  
优化后：
  作品弹窗 → 点击查看章节 → 弹窗内展开章节内容 → 点击关闭 → 回到作品弹窗
```

#### P3-04: Tab 切换状态保持

**问题**：从"创作"切换到"作品"再切回来，填写的所有表单数据丢失。

**方案**：
- 将 `storyForm`、`charactersList`、`foreshadowings` 等状态保存到 `sessionStorage`
- Tab 切换时自动保存快照，回到创作 tab 时恢复
- 提供"清空表单"按钮以重置状态

### 第二优先级：P1（核心体验增强）

#### P3-14: 卷与章节的拖拽排序

**目标**：支持卷的拖拽排序和章节跨卷拖拽。

**方案**：
- 使用 HTML5 Drag & Drop API（无需第三方库，保持 CDN 方式）
- 拖拽时显示占位线指示目标位置
- 卷内章节拖拽排序，卷间章节迁移

**交互设计**：
```
卷 1 [拖拽手柄]
  ├── 第1章 [拖拽手柄]
  ├── 第2章 [拖拽手柄]
  └── 第3章 [拖拽手柄]
  
卷 2 [拖拽手柄]
  ├── 第4章 [拖拽手柄]
  └── ...
```

**后端 API 扩展需求**：
- `PUT /works/{id}/volumes/reorder` — 卷排序
- `PUT /works/{id}/volumes/{vid}/chapters/reorder` — 章节排序
- `PUT /works/{id}/chapters/{cid}/move` — 章节跨卷移动

#### P3-02: Toast 系统升级

**方案**：
- 引入 Toast 队列管理，支持同时多条通知
- 增加动画效果（slide-in from right）
- 按类型区分位置：成功/信息显示在右上，错误显示在右下
- 支持手动关闭（点击 toast）
- 错误 toast 自动展示更多详情（可展开）

#### P3-12: 作品列表增强

**方案**：
- 卡片信息增加：字数统计、最后更新时间、卷数/章节数
- 添加搜索框（按标题搜索）
- 添加过滤筛选（按体裁/风格）
- 列表/卡片视图切换
- 支持按更新时间/字数排序

#### P3-08: 生成过程进度细化

**方案**：
- 将生成过程分为 4 个阶段显示进度条
  ```
  [策划中...] → [角色构建中...] → [写作中...] → [润色中...]
  [████████░░░░░░░░░░░░] 45%
  ```
- 显示当前智能体的输出片段（已有 SSE 数据流，优化展示方式）
- 生成完成后自动停止进度动画

## 4. 参考设计原则

基于搜索得出的设计模式研究，本方案遵循以下原则：

1. **费茨定律**：高频按钮（生成、保存）应置于易触区域，适当放大
2. **渐进式展示**：复杂操作分步呈现，不一次性展示所有选项
3. **容错设计**：危险操作提供二次确认 + 可配置的撤销机制
4. **状态可见性**：用户始终知道"我在哪、能去哪、怎么返回"
5. **一致性**：所有弹窗遵循相同的关闭模式（ESC + 点击遮罩 + X 按钮）

**参考来源**：
- Vue 3 模态组件设计模式（Teleport + Transition + 焦点管理）
- 创作工具 UX（Scrivener/WriteO/NovelPad 的章节管理和拖拽交互）
- App UI 设计黄金法则（反馈机制、动画过渡、一致性）

```
  ├── 快捷键 + 骨架屏（效率提升）

```

## 6. 技术实现说明

所有优化均在现有技术栈内实现：
- **Vue 3 Composition API**（CDN 方式）
- **原生 HTML5 API**（Drag & Drop、localStorage、sessionStorage）
- 不引入额外第三方库，保持 CDN 轻量方式
- 后端仅需为拖拽排序新增少量 API 端点
