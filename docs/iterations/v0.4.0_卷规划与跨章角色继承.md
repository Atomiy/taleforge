# v0.4.0 卷规划与跨章角色继承

## 概述
本次迭代实现了**卷级大纲规划系统**，解决了续写时角色状态不跨章传递的核心问题，并完成了第三部作品「深渊回响」的完整三章生成验证。

## 改动清单

### 1. Volume 模型扩展
**文件**: [backend/models/work.py](file:///c:/Users/33173/Desktop/study/taleforge/backend/models/work.py)

| 新增字段 | 类型 | 说明 |
|:---|:---|:---|
| `outline` | str | 卷故事大纲（含每章简述） |
| `responsibility` | str | 本卷在整部作品中的职责 |
| `connection_to_prev` | str | 与上一卷的衔接说明 |
| `connection_to_next` | str | 对下一卷的铺垫说明 |
| `characters` | List[dict] | 本卷出场人物（name/identity/brief） |
| `foreshadowings` | List[str] | 本卷伏笔清单 |

### 2. StoryRequest 模型扩展
**文件**: [backend/models/story.py](file:///c:/Users/33173/Desktop/study/taleforge/backend/models/story.py)

新增字段：`work_id`, `volume_id`, `chapter_number` — 使生成请求能关联到具体作品和卷。

### 3. 卷规划注入生成流程（三层传递）
**文件**: [backend/services/orchestrator.py](file:///c:/Users/33173/Desktop/study/taleforge/backend/services/orchestrator.py)

- orchestrator 启动时根据 `request.work_id` + `request.volume_id` 加载卷规划
- `self._volume_plan` 保存为实例变量，确保后续各 agent 都能访问
- 三处注入点：
  - **Planner**: `context["volume_plan"]` — 提示词中标注"卷规划优先级高于所有其他信息"
  - **Character**: 卷规划人物合并到 `existing_characters`，跳过已存在角色
  - **Writer**: 通过 `volume_plan` 参数传入，提示词中标注"优先级高于本章大纲"

### 4. Planner 提示词改进
**文件**: [backend/agents/planner.py](file:///c:/Users/33173/Desktop/study/taleforge/backend/agents/planner.py)

- 新增 `volume_plan_info` 区块，显示卷职责、大纲、出场人物、伏笔
- 明确标注"卷规划优先级高于所有其他信息"
- 生成的章节必须服务于本卷的职责和进度

### 5. Character Agent 继承卷规划角色
**文件**: [backend/services/orchestrator.py](file:///c:/Users/33173/Desktop/study/taleforge/backend/services/orchestrator.py)

### 6. Writer 提示词加入卷规划
**文件**: [backend/agents/writer.py](file:///c:/Users/33173/Desktop/study/taleforge/backend/agents/writer.py)

- `_build_system_context` 新增 `volume_plan` 参数

- 标注"本章内容必须服务于本卷的职责和进度"

## 修复的 Bug

| Bug | 原因 | 修复 |
|:---|:---|:---|
| 灵感按钮无响应 | `getInspiration` 等未从 Vue 3 setup() return | 补全 return 语句 |
| 卷创建时 `volumes` 为空 | Work 模型 `sync_volumes` 仅在已有章节时创建默认卷 | 创建作品后显式调用 addVolume |
| Planner 报 `work_id` 属性错误 | StoryRequest 模型缺少 work_id/volume_id 字段 | 模型扩展 |
| 各章角色不一致 | character agent context 重建时丢失 volume_plan | 改为 `self._volume_plan` 实例变量 |

## 验证结果

### 第三部作品「深渊回响」（克苏鲁+硬科幻）
| 章节 | 标题 | 字数 | 角色一致性 |
|:---|:---|:---|:---|
| 第1章 | 深渊低语 | 2796 | 林深、陈霜、索菲娅、赵海、深渊之眼 |
| 第2章 | 隐藏信息 | 4051 | 同上 ✅ |
| 第3章 | 同步频率 | 2603 | 同上 ✅ |
| **合计** | | **9450** | |

### 章节衔接质量（用户反馈）
| 维度 | 评分 | 说明 |
|:---|:---:|:---|
| 章节间物理衔接 | 9.5/10 | 首句、场景、角色状态无懈可击 |
| 情节进度连续性 | 9.0/10 | 每章有新信息，且不可逆 |
| 核心冲突递进 | 8.5/10 | 升级路径清晰（争论→密谋→失控） |
| 角色状态连贯 | 9.0/10 | 林深的心理状态贯穿三章，层层加深 |
| **综合** | **9.0/10** | 优于苍狼传说基准，远胜星际余晖旧版 |

### 对比验证
- **因果驱动 vs 时间驱动**：第2章结尾"七环符号与心跳同步"直接触发第3章全部剧情，因果链连续且紧密
- **进度锚点有效**：第3章开头紧接第2章结尾状态（黑暗、心率138），无时间回退
- **卷规划继承**：5个角色跨章一致，planner 收到的大纲提示确实锚定了章节定位
