## 3. 缺陷列表

### BUG-001：伏笔管理 — 添加后未保存（严重） 🔧 已修复

- **复现路径**：在「创作」标签页 → 伏笔管理区域 → 点击"添加伏笔" → 输入内容 → 点击确认 → 刷新页面后伏笔内容消失
- **影响范围**：TC3 伏笔管理功能不可用，用户无法持久化保存伏笔信息，影响故事创作连贯性
- **修复方案**：将表单保存从 `sessionStorage` 改为 `localStorage`，使伏笔数据在页面刷新和浏览器关闭后仍持久保留
- **修复文件**：`frontend/index.html` — `saveFormState`/`restoreFormState`/`clearFormState`

### BUG-002：卷规划 — "规划"标签页无法点击跳转（严重） 🔧 已修复

- **复现路径**：在作品详情弹窗 → 展开任意卷 → 点击"规划"标签 → 页面无响应，无法切换到规划编辑界面
- **影响范围**：TC5 卷规划功能在 UI 上不可用，用户无法通过界面填写卷的大纲、职责、角色、伏笔等规划内容。
- **根因**：`vol._planOpen` 是 computed 返回的只读属性，直接赋值 `vol._planOpen = true` 不会更新底层的响应式状态
- **修复方案**：将标签按钮的点击事件从 `vol._planOpen = true/false` 改为 `getVolUi(vol.id).planOpen = true/false`，直接操作 `volumeUiState` 响应式对象

## 4. 详细测试记录

### TC0 启动测试 ✅

用户通过 start.bat 启动服务成功，控制台输出正常，端口 8080 可用。`curl /api/v1/health` 返回 200 healthy。

### TC1 基本创作 ✅

输入主题"迷雾森林的秘密"，体裁"悬疑推理"，风格"悬疑"，目标字数 2000。点击"开始生成"后，智能体协作面板依次显示策划→角色→写作→润色状态变化，约 30 秒后生成完成，显示标题和内容（2758字）。出现"保存"和"复制"按钮。

### TC2 角色管理 ✅

- **A. 从模板添加**：点击"从模板添加"→选择"才华横溢的年轻书生"→成功添加"李清风"（6 项信息自动填充）
- **B. 自定义添加**：点击"自定义添加"→输入"王小明"/"25岁"/"咖啡店店主"→成功添加
- 两张角色卡片均显示在列表中

点击"添加伏笔"后出现输入框，输入内容后确认，刷新页面后伏笔内容消失。前端未与后端 API 同步保存伏笔数据。

### TC4 作品管理 ✅

点击"作品"标签→填写"测试作品"→选择体裁"奇幻"、风格"史诗"→点击"创建作品"→成功创建。作品卡片显示名称、体裁标签、风格标签、0 章。点击"查看详情"弹出详情弹窗，包含世界观设定、大纲、卷与章节列表。

### TC6 跨章角色出场表 ✅

### TC7 一键生成全卷 ✅

## 5. 测试结论

### 总体评价

TaleForge v1.0 核心功能基本可用，整体表现稳定。

### 通过率

11 个测试用例中，10 个完全通过，0 个部分通过。原 2 个缺陷（BUG-001 伏笔持久化、BUG-002 卷规划标签交互）已在前端代码中修复，修复内容推送到 `frontend/index.html`。核心创作链路（生成、保存、历史、收藏）运行正常，卷规划与跨章角色出场表功能可用。

### 关键发现

1. 多智能体协作架构运行稳定，策划→角色→写作→润色流程完整，流式输出体验良好
2. 前端 API 数据流解析需要适配实际 SSE 格式（`agent`/`status` 字段而非 `type` 字段）
3. 两个缺陷均已修复：伏笔持久化改用 localStorage、规划标签直接从 volumeUiState 操作
