# TaleForge 用户使用指南

## 快速开始

### 1. 启动项目

```bash
cd C:\Users\33173\Desktop\study\taleforge
python start.py
```

启动器将自动检测 8080 端口并启动服务器，打开浏览器访问 `http://127.0.0.1:8080`。

> **首次使用**：需要先在页面右上角点击"设置"按钮，配置 DeepSeek API Key。

### 2. 生成第一个故事

1. 在"故事主题"输入框中输入主题（如"勇者屠龙"）
2. （可选）选择体裁、风格、目标字数
3. 点击"开始生成"按钮
4. 观察智能体协作面板的实时状态
5. 生成完成后自动保存到历史记录

## 功能详解

### 创作页面

#### 基本设置
- **故事主题**：必填，描述故事的核心内容
- **体裁**：短篇小说、中篇小说、科幻、奇幻、悬疑推理等
- **风格**：温馨、悬疑、科幻、古风、奇幻、史诗等
- **目标字数**：拖动滑块设置 500~5000 字

#### 剧情设置
- **故事背景**：时代、地点、社会背景
- **叙事视角**：第三人称全知/有限、第一人称、第二人称
- **冲突类型**：人与自然、人与社会、人与人、内心冲突
- **情感基调**：轻松愉快、紧张刺激、温馨感人、深沉思考等
- **故事梗概**：大致的情节走向

#### 世界观设定
- 描述架空世界的规则、种族、魔法体系、历史等
- 启用后 AI 将严格遵循该世界观进行创作

#### 伏笔管理
- 添加伏笔项，如"古老的预言"、"神秘的信物"
- AI 会在故事中合理埋设和回收这些伏笔

#### 角色设定

**自定义添加**（推荐）：
1. 点击"自定义添加"按钮
2. 在弹出的表单中填写：角色名称、年龄、身份、外貌、性格、背景、动机
3. 点击"确认添加"

**从模板添加**：
1. 点击"从模板添加"按钮
2. 从 16 个预设角色模板中选择
3. 角色自动添加到列表，并可进一步编辑

角色字段说明：
| 字段 | 说明 | 示例 |
|------|------|------|
| 角色名称 | 角色名字 | 李清风 |
| 年龄 | 角色年龄 | 22岁 |
| 身份/职业 | 社会角色 | 年轻书生 |
| 外貌特征 | 外观描述 | 面容清秀，身着青衫 |
| 性格特点 | 性格描述 | 温文尔雅，正直善良 |
| 背景故事 | 身世经历 | 出身书香门第 |
| 角色动机 | 驱动角色的目标 | 考取功名，施展抱负 |

#### 创作模板
- 点击"使用模板"按钮，从 8 种预设模板中选择
- 模板自动填充体裁、风格、视角、冲突、背景和梗概

### 历史页面

#### 故事卡片
- 显示标题、体裁、风格、字数、创建时间
- 操作按钮：收藏 ⭐、导出 ⬇️、续写 ✏️、删除 🗑️

#### 收藏功能
- 点击故事卡片的星标按钮收藏故事
- 使用"已收藏"筛选器只显示收藏的故事

#### 搜索与筛选
- 支持按标题/内容关键词搜索
- 支持按体裁筛选
- 支持按收藏状态筛选

#### 分页导航
- 每页最多显示 12 条记录
- 支持翻页浏览

#### 故事详情与编辑
1. 点击故事卡片查看详细内容
2. 点击"编辑"按钮进入编辑模式
3. 修改故事内容后点击"保存修改"
4. 点击"取消编辑"放弃修改

#### 导出故事
- 点击导出按钮，自动下载 Markdown 文件
- 可在 API 中指定导出格式（markdown / text）

#### 续写故事
- 点击"续写"按钮，自动加载前作信息
- 切换到创作页面，可继续生成后续章节

### 设置

- **API Key 配置**：输入 DeepSeek API Key 并保存
- Key 持久化保存在浏览器本地存储（localStorage），每次生成请求自动携带
- 如需更换 Key，在设置中修改并重新保存即可
- **分享项目**：将项目发给组员时，对方只需在设置中填入自己的 API Key 即可使用，无需修改 `.env` 文件

## 五、高级功能（v0.4.0）

### 5.1 卷规划

作品详情页打开后，点击卷右侧的「规划」标签页，可以详细规划每一卷：

- **卷职责**：本卷在整部作品中的角色定位（如"引入核心冲突"、"推进高潮"）
- **卷故事大纲**：本卷的故事脉络和发展方向
- **卷间衔接**：与上卷的联系、对下卷的铺垫
- **本卷出场人物**：添加本卷所有角色的名称、身份、简述
- **本卷伏笔清单**：本卷需埋设和回收的伏笔

规划完成后点击「保存规划」，数据将持久化到作品文件。

### 5.2 跨章角色出场表

当一卷有多个章节时，可在规划标签页中为每章分配出场角色：

1. 先在「本卷出场人物」中添加所有角色
2. 在「跨章人物出场表」中，每章对应一组复选框
3. 勾选某角色 = 该角色在本章出场；取消勾选 = 不出场
4. ★ 标记表示某角色在本章**首次登场**
5. 点击「一键填充」可将所有角色分配到所有章节

系统会自动校验并显示警告（如角色从未出场、出场表中有不存在的角色）。

### 5.3 角色出场约束

作品数据可配置每章每角色的出场类型（通过后端数据或脚本定义）：

| 类型 | 含义 | 行为约束 |
|:---:|:-----|:---------|
| **主动** | 角色正常出场 | 可说话、行动、推进剧情 |
| **被动** | 角色被提及但不出声不行动 | 仅通过他人转述/数据/痕迹出现，无对话无动作 |
| **缺席** | 角色完全不出现 | 任何形式都不应出现其名 |

当某角色在特定章节为「被动」状态时，建议将该章核心驱动力设为「外部事件」（如环境变化、新发现、外部危机），而非该角色的内部状态——这就是**事件驱动设计**原则。

### 5.4 一键生成全卷

在作品详情页的卷头部，点击 ✨（一键生成全卷）按钮：

1. 系统读取该卷的 `chapter_characters` 章节角色表
2. 自动按章逐章生成，每章使用正确的角色列表
3. 自动链接前后章节（上一章的保存 ID 传递给下一章）
4. 生成过程中显示进度覆盖层（当前第 X/Y 章）
5. 全部完成后自动刷新作品详情，显示新生成的章节

> **注意事项**：如果该卷已有已保存章节，系统会提示"覆盖确认"。

### 5.5 自动填充

从作品详情点击「创作第一章」或「创作第N章」时，系统自动填充：

- **故事主题** ← 作品标题
- **世界观设定** ← 作品世界观
- **故事梗概（大纲）** ← 卷规划中的大纲
- **角色列表** ← 卷规划中的角色列表
- **伏笔** ← 作品级伏笔清单
- **章节序号** ← 自动递增（已有章节则续写模式）

这确保了从前端生成与从脚本生成的一致性。

## 常见问题

**Q: 生成失败怎么办？**
A: 检查 API Key 是否正确配置，以及网络是否可访问 DeepSeek API。如果使用一键生成全卷中途失败，可重新生成，系统会覆盖已有章节。

**Q: 端口被占用？**
A: 启动器会自动杀死占用 8080 端口的进程。如果自动处理失败，可手动关闭占用程序后重试。

**Q: 如何更新故事内容？**
A: 在历史记录中点击故事卡片查看详情，点击"编辑"按钮修改内容，然后"保存修改"。

**Q: 如何继续之前的故事？**
A: 在历史记录中点击故事的"续写"按钮，系统自动加载前作信息，可继续生成后续章节。也可以在作品详情中点击"续写"按钮。

**Q: 支持哪些故事体裁？**
A: 短篇小说、中篇小说、日常小短文、散文、剧本、童话、诗歌、悬疑推理、科幻、奇幻。

**Q: 一键生成全卷和逐个生成有什么区别？**
A: 一键生成自动遍历所有章节、链接上下文、无需手动操作。逐个生成适合调试和精细化控制——每生成一章后可以检查修改，再点"续写下一章"。

**Q: 如何确保角色不出现在不应出现的章节？**
A: 在卷规划的「跨章人物出场表」中，只勾选该章应出场的角色。未勾选的角色不会出现在该章的角色列表中，从而大幅降低"违规出场"的概率。

## 四、作品管理（新功能）

### 4.1 创建作品

点击导航栏的「作品」标签，在「创建新作品」区域填写：
- **作品名称**：你的史诗作品标题
- **体裁/风格**：选择适合的类型
- **世界观设定**：描述故事世界的背景规则
- **故事大纲**：规划整部作品的发展脉络

点击「创建作品」即可保存。

### 4.2 管理章节

作品详情页会按顺序显示所有章节。每章可以进行：
- **查看**：浏览章节完整内容
- **续写**：从该章继续生成下一章（自动编号）
- **删除**：移除某章

### 4.3 关联故事到作品

创作故事时，可以在「基本设置」区域选择「所属作品」下拉框。生成并保存的故事会自动添加到对应作品中。

### 4.4 生成作品概要

在作品详情页点击「生成概要」，系统会根据已有章节内容自动生成整部作品的概要。

### 4.5 章节编号

系统会自动检测前作内容的章节编号（如「第一章」「第二章」），续写时自动递增，保证编号连续不重复。
