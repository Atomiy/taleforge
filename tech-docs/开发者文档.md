## 核心数据模型

### Story（故事）

```python
class Story(BaseModel):
    id: str                    # 唯一标识
    title: str                 # 故事标题
    content: str               # 故事内容
    theme: str                 # 创作主题
    genre: str                 # 体裁
    style: str                 # 风格
    outline: Optional[Outline] # 故事大纲
    characters: Optional[List] # 角色列表
    created_at: datetime       # 创建时间
    word_count: int            # 字数
    series_id: Optional[str]   # 系列故事 ID
    series_order: int          # 系列顺序
    world_setting: Optional[str] # 世界观设定
    foreshadowings: Optional[List[str]] # 伏笔清单
    previous_story_id: Optional[str] # 前作 ID
    favorite: bool             # 收藏状态
```

### Character（角色）

```python
class Character(BaseModel):
    name: str                  # 名称
    age: str                   # 年龄
    identity: str              # 身份/职业
    appearance: str            # 外貌特征
    personality: str           # 性格特点
    background: str            # 背景故事
    motivation: str            # 角色动机/目标
    relationships: List        # 角色关系
    drive: str                 # 核心动机和恐惧
    contradiction: str         # 表面与内心的冲突
    arc: str                   # 角色成长弧线
```

## 多智能体协作流程

```
输入参数
     │
     ▼
┌─────────────┐
│   Planner   │  → 生成故事大纲（结构、阶段、章节、伏笔映射）
│  策划智能体  │
└──────┬──────┘
       │ 大纲数据
       ▼
┌─────────────┐
│  Character  │  → 深化角色设定（驱动、矛盾、弧线、对话风格）
│  角色智能体  │
└──────┬──────┘
       │ 角色数据
       ▼
┌─────────────┐
│   Writer    │  → 逐章写作（流式输出内容）
│  写作智能体  │
└──────┬──────┘
       │ 原始内容
       ▼
┌─────────────┐
│  Polisher   │  → 润色整合（修正、统一风格、最终输出）
│  润色智能体  │
└──────┬──────┘
       │ 最终故事
       ▼
    输出完成
```

## 数据流说明

1. **前端 → 后端**：用户填写表单 → `POST /api/v1/story/generate` → SSE 流式响应
2. **智能体调度**：Orchestrator 依次调用 Planner → Character → Writer → Polisher
3. **流式输出**：每个智能体的状态和阶段性结果通过 SSE 事件实时推送到前端
4. **自动保存**：生成完成后前端自动调用 `POST /api/v1/story/save`
5. **历史查询**：前端调用 `GET /api/v1/history/?page=1&page_size=12` 获取历史列表

## 扩展指南

### 添加新体裁
1. 在 `frontend/index.html` 的 `genres` 数组中添加体裁名称
2. 在创作模板（`storyTemplates`）中添加对应模板
3. AI 会在提示词中自动学习使用该体裁

### 添加新智能体
1. 在 `backend/agents/` 目录下创建新智能体模块
2. 在 `orchestrator.py` 的 `AGENT_ORDER` 列表中注册执行顺序
3. 在前端 `agents` 数组中添加对应卡片配置

### 添加新 API 路由
1. 在 `backend/routers/` 下创建或修改路由文件
2. 在 `main.py` 中注册路由（`app.include_router(router, prefix="/api/v1/...")`）

## 十、作品管理系统

### 10.1 数据模型

`Work` 模型（`backend/models/work.py`）：
- `id`：唯一标识符（UUID前8位）
- `title`：作品名称
- `world_setting`：世界观设定
- `foreshadowings`：伏笔列表
- `outline`：故事大纲
- `summary`：内容概要
- `chapter_ids`：章节ID有序列表（关联 Story）
- `genre` / `style`：体裁/风格
- `created_at` / `updated_at`：时间戳

### 10.2 存储方式

`WorkManager`（`backend/services/work_manager.py`）使用 JSON 文件存储，文件位于 `backend/data/works.json`。

### 10.3 API 端点

全部端点位于 `/api/v1/works/`，详见 API 文档。

### 10.4 章节编号机制

续写时，`orchestrator.py` 自动从上一篇故事内容中通过正则 `## 第(\\d+)章` 检测章节号，并递增后传入 Writer 智能体上下文。
